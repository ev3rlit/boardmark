# 백로그: 좌표 정수 스냅

## 문제

웹 UI에서 노트를 드래그하면 좌표가 부동소수점으로 저장된다.

```md
::: note { id: note-1, at: { x: 5994.3, y: -33.82, w: 717, h: 1157 } }
```

- 사람이 텍스트를 직접 읽거나 편집하기 어려움
- git diff 가 의미 없는 소수점 변경으로 가득 참
- AI가 좌표를 다룰 때 불필요한 노이즈가 됨

## 제안: 저장 시 정수 스냅

UI에서 드래그·리사이즈 후 직렬화할 때 좌표를 정수로 반올림한다.

```md
::: note { id: note-1, at: { x: 5994, y: -34, w: 717, h: 1157 } }
```

- 스냅 단위는 1px (정수) 또는 설정 가능한 그리드 단위 (예: 4, 8px)
- 렌더링 중 내부 상태는 부동소수점 유지, 직렬화 시점에만 스냅 적용

## 고려 사항

- 스냅 단위를 frontmatter 또는 설정으로 노출할지 여부
- 기존 파일에 소수점 좌표가 있어도 동작에는 영향 없음 — 파서는 그대로 수용하고 다음 저장 시 자동 정규화
