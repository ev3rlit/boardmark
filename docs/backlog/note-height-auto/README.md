# 백로그: note 높이 자동 계산 (`h` 생략 시)

## 문제

현재 note 오브젝트는 `h` 필드가 필수에 가깝다. 콘텐츠 양이 달라질 때마다 `h` 값을 수동으로 조정해야 하고, 맞추지 않으면 내용이 넘치거나 빈 공간이 생긴다.

- 웹 UI에서 마크다운 내용을 수정하면 `h` 가 즉시 어긋남
- AI가 note를 생성할 때 적절한 `h` 값을 예측하기 어려움
- 텍스트로 직접 작성할 때도 높이 추정이 번거로움

## 제안: `h` 생략 시 자동 계산

```md
::: note { id: intro, at: { x: 0, y: 0, w: 420 } }
내용에 따라 높이가 자동으로 결정된다.
:::

::: note { id: fixed, at: { x: 500, y: 0, w: 420, h: 280 } }
h를 명시하면 고정 높이로 처리한다.
:::
```

- `h` 생략 → 렌더링 후 콘텐츠 높이 기반으로 자동 계산
- `h` 명시 → 기존 동작 유지 (고정 높이, 넘치면 스크롤 또는 클리핑)

## 고려 사항

- 자동 계산 시 렌더링 전 레이아웃을 알 수 없으므로, 초기 렌더 후 리사이즈 발생 — 레이아웃 점프 방지 전략 필요
- 직렬화 시 계산된 `h` 를 파일에 다시 쓸지 여부 결정 필요
  - 쓴다면 → 파일이 자동으로 변경되어 git diff 노이즈 발생
  - 안 쓴다면 → 매 로드마다 재계산 필요
