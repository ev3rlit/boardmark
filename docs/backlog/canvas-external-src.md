# 백로그: 대용량 캔버스 파일 분리 (`src` 속성)

## 문제

캔버스 파일이 1000줄을 넘어갈 경우:

- AI 에이전트가 파일 전체를 컨텍스트에 올려야 해서 토큰 낭비 및 실수 가능성 증가
- 사람이 탐색하기 어려워짐
- git diff 충돌 빈도 증가

## 제안: `src` 속성으로 외부 파일 참조

```md
::: note
id: deep-dive
x: 200
y: 300
src: ./notes/deep-dive.md        ← .md: 콘텐츠를 외부 마크다운 파일로 위임
:::

::: group
id: subsystem
x: 600
y: 100
src: ./subsystem.canvas.md       ← .canvas.md: 다른 캔버스를 노드로 임베드
:::
```

- `.md` → 해당 파일의 내용을 노드 콘텐츠로 사용
- `.canvas.md` → 해당 캔버스를 그룹 노드로 임베드
- AI가 특정 노드만 편집할 때 해당 파일만 읽으면 됨
- 확장자 하나로 "이 노드의 실제 내용은 저 파일"이라는 의미를 전달

## 전제 조건

- `canvas-format-v2` 포맷 및 파서 안정화 이후 검토
